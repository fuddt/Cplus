# Windowsコードって何？

## このドキュメントの目的

このドキュメントは、

- `HWND`
- `DWORD`
- `WPARAM`
- `LPARAM`
- `MessageBox`
- `WM_COMMAND`

のような **Windowsのコード特有の単語** を初めて見た人向けに、

「そもそも Windowsコードって何なの？」

をわかるようにするためのものです。

対象は、**C++を少し学び始めた中学生くらい** を想定しています。

---

## まず結論

**Windowsコード** とは、

**Windows というOSの機能を使うためのコード**

のことです。

たとえばWindowsには、最初から次のような機能があります。

- ウィンドウを表示する
- ボタンを置く
- クリックされたことを知らせる
- 小さなメッセージ画面を出す
- キーボード入力を受け取る

Windowsコードは、こういう機能を使うために書くコードです。

つまり、Windowsコードは

- 食べ物ではない
- 生き物でもない
- 人の名前でもない

たとえるなら、**Windowsにお願いを出すための特別な言葉** です。

---

## C++とWindowsコードは同じもの？

同じではありません。

ここが一番大事です。

### C++
C++は **プログラミング言語** です。

たとえば、こういうものはC++です。

- `if`
- `class`
- `for`
- `std::vector`
- `std::unique_ptr`

### Windowsコード
Windowsコードは、**Windowsの機能を使うための書き方** です。

たとえば、こういうものはWindows側の言葉です。

- `HWND`
- `MessageBox`
- `WM_COMMAND`
- `DWORD`

つまり、

- **C++ = 言語そのもの**
- **Windowsコード = Windows向けの特別ルールつきのコード**

です。

---

## たとえ話

たとえば、C++を「日本語」だとします。

するとWindowsコードは、

**日本語で書かれた、Windows市役所向けの申請書**

みたいなものです。

日本語そのものを知っていても、市役所の書類の書き方は別に覚えないといけません。

それと同じで、

- C++を学んだ
- でもWindowsコードはまだ知らない

という状態は、まったく変ではありません。

---

## Windowsコードには何が出てくるの？

Windowsコードには、いろいろな種類のものが出てきます。

全部を「よくわからない記号の山」と思うと苦しくなります。

なので、まず **仲間分け** します。

---

## 1. 型

これは「どんな種類のデータか」を表す言葉です。

### 例
- `HWND`
- `DWORD`
- `UINT`
- `WPARAM`
- `LPARAM`
- `LRESULT`

### 覚え方
型は、C++でいう `int` や `double` や `std::string` の仲間です。

Windowsでは、Windows用の特別な型の名前がたくさんあります。

### ざっくり意味
- `HWND` = ウィンドウを指すための番号札みたいなもの
- `DWORD` = 整数の別名
- `UINT` = 符号なし整数
- `WPARAM` = 通知についてくる情報その1
- `LPARAM` = 通知についてくる情報その2
- `LRESULT` = Windowsに返す返事の型

ここで大事なのは、

**見慣れないだけで、全部が魔法の存在ではない**

ということです。

---

## 2. 関数

これは「やってほしい仕事を頼むもの」です。

### 例
- `MessageBox`
- `CreateWindow`
- `SetWindowText`

### ざっくり意味
- `MessageBox` = 小さなメッセージ画面を出して
- `CreateWindow` = ウィンドウやボタンを作って
- `SetWindowText` = 文字を変えて

つまり、Windowsの機能を呼び出しているだけです。

---

## 3. 通知メッセージ

Windowsでは、ボタンが押されたり、画面が閉じられたりすると、
アプリに「こういうことが起きたよ」と知らせてきます。

それが **メッセージ** です。

### 例
- `WM_CREATE`
- `WM_COMMAND`
- `WM_DESTROY`

### ざっくり意味
- `WM_CREATE` = 画面が作られたよ
- `WM_COMMAND` = ボタンが押されたよ、メニューが選ばれたよ
- `WM_DESTROY` = 画面が閉じられるよ

これはかなり大事です。

C# の WinForms だと「Clickイベント」が前に見えますが、
Windowsの世界ではもっと下のところで、

**メッセージが飛んでくる**

という形で動いています。

---

## 4. マクロ

マクロは、コンパイルする前にコードを書きかえるための仕組みです。

### 例
- `#define`
- `#ifdef`
- `#ifndef`

### ざっくり意味
- `#define` = あだ名や目印を作る
- `#ifdef` = ある目印があるなら、このコードを使う
- `#ifndef` = ある目印がないなら、このコードを使う

### 例
```cpp
#define APP_NAME L"InventoryApp"

#ifdef _DEBUG
    MessageBox(nullptr, L"デバッグ中です", APP_NAME, MB_OK);
#endif
```

これは、

- `APP_NAME` という名前を作る
- `_DEBUG` があるときだけ、その部分を使う

という意味です。

---

## 5. 画面部品のID

ボタンや入力欄には、それぞれ番号や名前をつけます。

### 例
- `IDC_BUTTON1`
- `IDC_OPEN_MENU`
- `IDC_LIST_ITEMS`

これは、

**どの部品が押されたのかを見分けるための名札**

です。

たとえば、ボタンが3つあったとします。

- メニューを開く
- ダメージを与える
- アイテムを追加する

そのとき、どのボタンが押されたのかを区別するためにIDを付けます。

---

## Windowsコードの本当の中心

Windowsコードの中心は、たった4つです。

1. 画面を作る
2. ボタンなどの部品を置く
3. 何か操作されたら通知を受ける
4. その通知に合わせて処理する

これだけです。

見た目は難しそうでも、やっていること自体は単純です。

---

## たとえば、ボタンが押されたら何が起きるの？

イメージだけ説明します。

1. 画面にボタンを置く
2. ボタンにIDをつける
3. ユーザーがボタンを押す
4. Windowsが「ボタンが押されたよ」と知らせる
5. プログラムがそれを見て処理する

つまり、

**ボタンが押されたから処理する**

という流れです。

---

## 最小のサンプル

次のコードは、ボタンが押されたらメッセージを出す流れの一部です。

```cpp
#include <windows.h>

#define IDC_BTN_TEST 1001

LRESULT CALLBACK WndProc(HWND hwnd, UINT msg, WPARAM wParam, LPARAM lParam)
{
    switch (msg)
    {
    case WM_COMMAND:
        if (LOWORD(wParam) == IDC_BTN_TEST)
        {
            MessageBox(hwnd, L"押された", L"通知", MB_OK);
        }
        return 0;
    }

    return DefWindowProc(hwnd, msg, wParam, lParam);
}
```

### 今はここだけ見ればよい
- `WM_COMMAND` = 何かの操作があった
- `IDC_BTN_TEST` = テスト用ボタンの名札
- `MessageBox` = 小さなメッセージ画面を出す

全部を理解しなくていいです。

まずは、

**Windowsは操作をメッセージとして知らせてくる**

これだけつかめば十分です。

---

## じゃあ、これまで学んだC++は無駄だったの？

無駄ではありません。

ここは大事です。

今まで学んだC++は、たとえば次のところで役に立ちます。

- クラスを作る
- アイテムを表す
- プレイヤーを表す
- インベントリを `vector` で持つ
- `unique_ptr` で所有権を管理する
- 関数を分ける

Windowsコードは、その上に

- 画面を表示する
- ボタンを置く
- クリックを受け取る

という役目が追加されただけです。

つまり、

- **C++ = アプリの中身を作る力**
- **Windowsコード = 画面や操作を扱う力**

です。

両方必要ですが、同じものではありません。

---

## 今の不安はおかしい？

おかしくありません。

初めてWindowsコードを見ると、

- 英単語が多い
- 大文字ばかりで怖い
- 何が型で何が関数かわからない
- 普通のC++と見た目が違いすぎる

ので、不安になるのは自然です。

でも、それは

**自分がダメだから** ではなく、
**まだWindowsという新しい分野の地図を持っていないから**

です。

これはかなり違います。

---

## 最初に覚えるべき最重要セット

全部いっぺんに覚える必要はありません。

最初はこの10個くらいで十分です。

- `HWND`
- `DWORD`
- `UINT`
- `WPARAM`
- `LPARAM`
- `LRESULT`
- `MessageBox`
- `WM_COMMAND`
- `#define`
- `#ifdef`

このへんが見えてくると、案件コードの「何もわからない」がかなり減ります。

---

## いまの段階でやってはいけないこと

### 1. `windows.h` を全部読もうとする
無理です。範囲が広すぎます。

### 2. 単語だけ丸暗記する
意味のつながりがわからないと、すぐ崩れます。

### 3. C++とWindowsコードを同じものだと思う
ここを混ぜるとずっと苦しいです。

---

## 今の自分をどう考えればいい？

今の自分を正しく言うなら、こうです。

- C++の基礎を学んでいる
- でもWindowsデスクトップアプリの文化はまだこれから
- だから見慣れない単語が多くて混乱している

これは自然な状態です。

「これまでの学習が無駄だった」ではありません。

ただ、次の段階に進むために、

**Windowsコードという新しいジャンルを別で学ぶ必要がある**

というだけです。

---

## まとめ

Windowsコードとは、

**Windowsの機能を使うためのコード**

です。

そこには、

- 型
- 関数
- 通知メッセージ
- マクロ
- 画面部品のID

がたくさん出てきます。

最初は怖く見えますが、仲間分けして見れば少しずつ読めます。

一番大事なのは、

**C++とWindowsコードは同じものではない**

と理解することです。

C++の基礎を学んできたことは無駄ではありません。
それは、Windowsデスクトップアプリの「中身」を作るために、そのまま使えます。

これから学ぶのは、

**Windowsの画面と操作の世界**

です。

---

## 次に読むとよいもの

このあと学ぶとよい順番は次の通りです。

1. `typedef` と `using`
2. `#define` と `#ifdef`
3. `HWND`, `DWORD`, `WPARAM`, `LPARAM`
4. `MessageBox`, `CreateWindow`
5. `WM_COMMAND`, `WM_CREATE`, `WM_DESTROY`
6. ボタン1個の最小Windowsアプリ

この順番でいけば、Windowsコードは「謎の文字列の山」ではなくなります。
