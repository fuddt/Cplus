# Windowsコード用語図鑑

地図ドキュメント `windows_code_for_junior_high.md` の補助資料です。

用語を1個ずつ、**何の仲間か** を先に示してまとめています。

---

## この図鑑の読み方

各用語は次の形式で書いています。

```
## 用語名
仲間: [型 / 関数 / メッセージ / マクロ / ID]
一言: 〜

```

---

## 型の仲間

型とは「どんな種類のデータか」を示す言葉です。
C++でいう `int` や `double` と同じ役割をします。

---

### HWND

**仲間:** 型
**一言:** ウィンドウを指す番号札
**使う場面:** ウィンドウやボタンを操作するとき、相手を指定するために使う

```cpp
HWND hwnd;  // このウィンドウを指す
```

「どのウィンドウか」を区別するための番号みたいなものです。
本やロッカーに番号がついているのと同じ感覚で使います。

---

### DWORD

**仲間:** 型
**一言:** 大きな整数の別名
**使う場面:** Windowsの関数が返す数値や、フラグをまとめた値を入れるとき

```cpp
DWORD style = WS_OVERLAPPEDWINDOW;
```

実体は `unsigned long` です。
C++の `int` に似ていますが、負の数を持たない大きな整数です。

---

### UINT

**仲間:** 型
**一言:** 符号なし整数の別名
**使う場面:** メッセージの種類番号や、IDを受け取るとき

```cpp
UINT msg;  // どんな通知が来たか
```

実体は `unsigned int` です。

---

### WPARAM

**仲間:** 型
**一言:** 通知についてくる情報その1
**使う場面:** `WM_COMMAND` のとき、どのボタンが押されたかを調べる

```cpp
WPARAM wParam;  // 情報その1
```

名前の由来は「Word Parameter」です。
`WM_COMMAND` では `LOWORD(wParam)` でボタンのIDが取れます。

---

### LPARAM

**仲間:** 型
**一言:** 通知についてくる情報その2
**使う場面:** クリックされた座標など、追加情報が入る

```cpp
LPARAM lParam;  // 情報その2
```

名前の由来は「Long Parameter」です。
場面によって意味が変わります。

---

### LRESULT

**仲間:** 型
**一言:** Windowsに返す返事の型
**使う場面:** `WndProc` という関数の戻り値の型として使う

```cpp
LRESULT CALLBACK WndProc(HWND hwnd, UINT msg, WPARAM wParam, LPARAM lParam)
{
    // ここで処理して、Windowsに返事を返す
    return 0;
}
```

「ちゃんと処理できたよ」などをWindowsに知らせます。

---

## 関数の仲間

関数とは「Windowsにやってほしい仕事を頼む言葉」です。
呼び出すと、Windowsが実際に動いてくれます。

---

### MessageBox

**仲間:** 関数
**一言:** 小さなメッセージ画面を出す
**使う場面:** エラー通知や確認ダイアログを表示したいとき

```cpp
MessageBox(hwnd, L"ボタンが押されました", L"通知", MB_OK);
```

引数の意味:
- 第1引数: 親ウィンドウ (`nullptr` でも動く)
- 第2引数: 本文
- 第3引数: タイトルバーの文字
- 第4引数: ボタンの種類 (`MB_OK` はOKボタン1個)

---

### CreateWindow

**仲間:** 関数
**一言:** ウィンドウやボタンを作る
**使う場面:** アプリ起動時にウィンドウを用意するとき、ボタンを置くとき

```cpp
HWND hBtn = CreateWindow(
    L"BUTTON",     // 部品の種類
    L"押してね",   // ボタンのラベル
    WS_CHILD | WS_VISIBLE | BS_PUSHBUTTON,
    10, 10, 100, 30,  // 位置とサイズ
    hwnd,          // 親ウィンドウ
    (HMENU)IDC_BTN_TEST,  // ID
    nullptr, nullptr
);
```

引数が多くて怖く見えますが、パターンはほぼ固定です。

---

### SetWindowText

**仲間:** 関数
**一言:** ウィンドウやボタンの文字を変える
**使う場面:** タイトルバーの文字を変えたいとき、ボタンのラベルを更新したいとき

```cpp
SetWindowText(hwnd, L"新しいタイトル");
```

---

### DefWindowProc

**仲間:** 関数
**一言:** 自分で処理しないメッセージをWindowsに任せる
**使う場面:** `WndProc` の末尾に必ず書く

```cpp
return DefWindowProc(hwnd, msg, wParam, lParam);
```

「このメッセージは自分では処理しないので、Windowsに任せます」という意味です。
書き忘れると、ウィンドウが正しく動かなくなります。

---

## メッセージの仲間

Windowsでは、操作があったときに「こういうことが起きたよ」という通知が飛んできます。
これをメッセージと呼びます。
`WM_` で始まる名前が目印です。

---

### WM_CREATE

**仲間:** メッセージ
**一言:** ウィンドウが作られたよ
**使う場面:** ウィンドウが表示される直前に1回だけ呼ばれる。ボタンを置くのはここ

```cpp
case WM_CREATE:
    // ここでボタンや部品を作る
    return 0;
```

---

### WM_COMMAND

**仲間:** メッセージ
**一言:** ボタンが押されたよ、またはメニューが選ばれたよ
**使う場面:** ボタンの押下を受け取って処理するとき

```cpp
case WM_COMMAND:
    if (LOWORD(wParam) == IDC_BTN_TEST)
    {
        // テストボタンが押された
    }
    return 0;
```

どのボタンかは `LOWORD(wParam)` で取れます。

---

### WM_DESTROY

**仲間:** メッセージ
**一言:** ウィンドウが閉じられるよ
**使う場面:** アプリ終了時の後片付け

```cpp
case WM_DESTROY:
    PostQuitMessage(0);  // アプリを終了する
    return 0;
```

`PostQuitMessage(0)` を書かないと、ウィンドウを閉じてもアプリが終わらないことがあります。

---

## マクロの仲間

マクロは、コンパイルより前にコードを書き換えるための仕組みです。
`#` で始まる行が目印です。

---

### #define

**仲間:** マクロ
**一言:** あだ名や目印を作る
**使う場面:** 数字に名前をつけるとき

```cpp
#define IDC_BTN_TEST 1001
#define APP_NAME L"InventoryApp"
```

`IDC_BTN_TEST` と書くだけで `1001` として扱われます。
数字のまま書き続けると「1001って何のボタン？」とわからなくなるので、名前をつけます。

---

### #ifdef / #endif

**仲間:** マクロ
**一言:** 「もしこの目印があるなら、このコードを使う」
**使う場面:** デバッグ用のコードを本番では含めないようにするとき

```cpp
#ifdef _DEBUG
    MessageBox(nullptr, L"デバッグ中", L"確認", MB_OK);
#endif
```

`_DEBUG` はデバッグビルドのときだけ自動で定義される目印です。
リリースビルドでは `MessageBox` の行がまるごと無視されます。

---

### #ifndef / #endif

**仲間:** マクロ
**一言:** 「もしこの目印がないなら、このコードを使う」
**使う場面:** ヘッダファイルが2重に読み込まれないようにするとき

```cpp
#ifndef MY_HEADER_H
#define MY_HEADER_H

// ヘッダの中身

#endif
```

「まだ読んでいないなら読む。もう読んでいるならスキップする」という仕組みです。

---

## IDの仲間

IDとは、ボタンや入力欄など「画面の部品を区別するための番号や名前」です。

---

### IDC_〇〇 (画面部品のID)

**仲間:** ID
**一言:** どの部品かを区別するための名札
**使う場面:** `WM_COMMAND` でどのボタンが押されたかを確認するとき

```cpp
#define IDC_BTN_OPEN   1001
#define IDC_BTN_ATTACK 1002
#define IDC_BTN_ADD    1003
```

ボタンが3個あるなら、3個それぞれに別のIDをつけます。
「1001番が押されたか？ 1002番が押されたか？」と確認します。
